

# BRO 予約システム 将来検討事項まとめ
（04_future_ideas.md）

## 1. 本ドキュメントの目的

本ドキュメントは、  
**MVPでは実装しないが、将来的に検討・実装する可能性が高い事項** を整理し、  
要件・設計・実装のブレを防ぐことを目的とする。

- 「今やらないこと」を明確にする
- 将来の議論をスムーズにする
- MVPのスコープを守るための防波堤にする

---

## 2. 決済まわりの検討事項

### 2.1 決済方式の選択肢

- クレジットカード決済
- 現地決済
- 事前決済（デポジット）
- 全額前払い

### 2.2 実装パターン候補

| パターン | 内容 |
|---|---|
| A | 完全人力（LINE / 電話で案内） |
| B | 外部決済サービスへ誘導（Square / Stripe 等） |
| C | システム内で完結（Webhook 連携） |

※ MVPでは **A または B** を想定  
※ C は中長期検討

---

## 3. 会員・リピーター管理

### 3.1 やりたいこと

- 過去予約の一覧を見せたい
- 名前・電話番号・メールの自動入力
- 問題のある顧客の管理

### 3.2 検討事項

- 会員登録あり / なし
- 会員IDを発行するか
- ログイン方式（メール / SMS / OAuth）

---

## 4. キャンセル規定・ペナルティ

### 4.1 キャンセルルール案

- 前日まで無料
- 当日キャンセルは100%
- 無断キャンセルは出禁

### 4.2 支払い方法の課題

- キャンセル料をどう請求するか
- 未払い時の対応
- 出禁処理（問い合わせ不可にする）

---

## 5. ステータス拡張案

現状：
- pending
- approved
- canceled

将来候補：
- rejected（不可・理由あり）
- expired（時間切れ）
- no_show（来店なし）
